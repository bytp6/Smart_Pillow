Compatibility Matrix: embARC Examples with EMSK 2.2

This EMSK version supports up to four configurations. The configurations have different hardware parameters such as memory size, core, and peripherals. The embARC examples can't always be compiled successfully for some configurations and optimization levels.

The following table provides compatibility data for compilation of all embARC examples with different configurations and optimization levels. The first column is the project name. The second column is the project path. The third to fifth columns are the supported optimization levels for the arcem7d, arcem9d, and arcem11d configurations. "xx" means that the specified example cannot be built successfully for the specified configuration.
